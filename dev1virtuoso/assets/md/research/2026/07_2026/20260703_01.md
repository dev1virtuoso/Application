## Abstract

This technical report explores innovative approaches to leveraging Large Language Models (LLMs) for semantic encryption, where plaintext messages are mapped to seemingly unrelated plaintext outputs to achieve covert communication. The core concept, exemplified by transforming "I like apple" into "The sky is blue," draws from steganography and cryptography principles. We propose creative yet practical ideas to address key challenges: poor determinism, complex key management, and weak security, while enhancing strengths like high concealment and strong pseudo-randomness. These ideas integrate established technologies such as Fully Homomorphic Encryption (FHE), Low-Rank Adaptation (LoRA) fine-tuning, Multi-Party Computation (MPC), and steganographic techniques. Drawing from recent literature, including works on generative text steganography and encrypted LLM inference, this report provides detailed implementation steps, benefits, challenges, and feasibility assessments. The report is structured into background, proposed solutions, evaluations, and future directions, aiming to guide researchers and practitioners in developing robust LLM-based encryption systems. this document synthesizes academic insights with practical engineering considerations.[^1][^2]

## Introduction

### Overview of Semantic Encryption with LLMs

Semantic encryption using LLMs represents a paradigm shift in cryptography, moving beyond traditional bit-level manipulations to high-level semantic mappings. In conventional encryption like AES, data is transformed into ciphertext that appears random and indecipherable without the key. However, in LLM-based semantic encryption, the output remains natural language plaintext but semantically unrelated to the input, blending into everyday communication channels like chats or social media posts. This approach is particularly suited for steganography—hiding the existence of the message itself—rather than just obscuring its content.

The foundational idea is to prompt an LLM to generate an output sentence or paragraph that encodes the input message via a controlled mapping process. For instance, using a prompt like "Encrypt 'I like apple' into an unrelated sentence with key 'secret123'" might yield "The sky is blue," which can be decoded with the reverse prompt. This leverages the LLM's generative capabilities for pseudo-randomness, making detection difficult without knowledge of the mapping scheme.

This report builds on a proposal of creative ideas that address inherent limitations of LLMs in this context: their stochastic nature leading to poor determinism, the complexity of managing model parameters as keys, and the lack of mathematical security proofs. By integrating technologies like blockchain for seed generation, MPC for key distribution, and FHE for secure computation, we aim to make these systems feasible for real-world applications such as secure messaging apps or covert data exfiltration.

### Motivation and Relevance

The rise of LLMs, such as GPT variants and open-source models on Hugging Face, has democratized advanced natural language processing. However, their application in security domains remains underexplored. Traditional steganography in text is limited by capacity and detectability, but LLMs offer higher embedding rates and more natural outputs. Recent events, including data breaches in AI systems, underscore the need for privacy-preserving techniques. For example, encrypted inference on LLMs prevents model owners from accessing user inputs, which is crucial for compliance with regulations like GDPR.[^3]

In 2025, with advancements in post-quantum cryptography and AI safety, LLM-based encryption aligns with trends toward hybrid crypto-AI systems. This report's proposals are grounded in feasibility, using open-source tools like web3.py for blockchain integration and Zama's Concrete-ML for FHE.[^4]

## Background and Literature Review

### Fundamentals of LLMs and Steganography

Large Language Models (LLMs) are transformer-based architectures trained on vast datasets to generate human-like text. Models like GPT-4 or Llama-3 exhibit emergent behaviors, including the ability to perform encoded reasoning or steganographic tasks. Steganography, the art of concealing information within non-secret carriers, has evolved from image-based methods to text-based ones. Generative text steganography uses LLMs to embed secrets in coherent narratives, achieving high imperceptibility.[^5][^6][^7]

Key challenges include maintaining semantic coherence while maximizing payload capacity. Recent works demonstrate LLMs' potential for black-box steganography, where users interact via APIs without model access.[^8]

### Related Works on LLM Encryption and Steganography

A paper, "Generative Text Steganography with Large Language Model" (2024), introduces LLM-Stega, a method for embedding bits into LLM-generated text using user prompts. This achieves embedding rates of up to 4 bits per sentence with low detection rates.[^9]

"Robust Steganography from Large Language Models" (2025) addresses robustness against paraphrasing attacks, defining weak and strong robustness metrics. It proposes fine-tuning LLMs to resist modifications.[^10]

In cryptography, "An LLM Framework For Cryptography Over Chat Channels" (2025) proposes embedding encrypted messages in LLM-generated chats for covert public communication.[^11][^12]

For FHE integration, "Towards Encrypted Large Language Models with FHE" (2023) demonstrates running transformer layers on encrypted data using Hugging Face libraries. "Encryption-Friendly LLM Architecture" (2024) modifies transformers for efficient homomorphic operations.[^13][^14]

LoRA fine-tuning is explored in "Practical Tips for Finetuning LLMs Using LoRA" (2023), showing how it reduces randomness for deterministic outputs. "Activated LoRA: Fine-tuned LLMs for Intrinsics" (2025) applies LoRA for conditional outputs in encryption scenarios.[^15][^16]

MPC in key generation is detailed in "Secure multi-party computation" (Wikipedia) and "Multiparty Computation (MPC) for Agile and Secure Enterprise Key Management" (2024), highlighting distributed key creation without single-point failures.[^17][^18]

Blockchain for randomness is covered in "Secure Random Number Generation in Blockchain Environments" (2024), using block hashes as entropy sources.[^19]

Steganography in chats: "Multi-criteria linguistic optimization for covert communication in LLMs" (2025) optimizes embeddings for naturalness.[^20]

Differential steganography: "LLMs Unleash the Power of Detecting Steganography" (2024) discusses multi-layer approaches.[^21]

Context-adaptive generation: "Improved Visual Story Generation with Adaptive Context Modeling" (2023) adapts stories to contexts for coherence.[^22]

Community-driven models: "The AI Community Building the Future? A Quantitative Analysis of Hugging Face" (2024) analyzes model sharing for diversity.[^23]

### Gaps in Existing Research

While existing works focus on basic embedding, they lack integration with advanced crypto primitives for determinism and security. This report fills these gaps with hybrid ideas.

## Proposed Ideas

### Addressing Poor Determinism

#### Blockchain-Synchronized Dynamic Seed Generation

To mitigate LLM stochasticity, we propose using blockchain events for shared seed generation. Blockchains provide verifiable, tamper-resistant randomness via hashes.[^24][^25]

**Implementation Steps:**

1. Parties agree on a smart contract monitoring events (e.g., Ethereum block hashes).

2. Encryption: Compute seed = hash(key + event_hash); prompt LLM with fixed seed for reproducible output.

3. Decryption: Replicate seed using the same event.

**Benefits:** Enhances concealment by avoiding seed transmission; synchronous randomness improves reliability.

**Challenges:** Network latency; mitigated by caching recent blocks.

**Feasibility:** Implementable with web3.py; suitable for decentralized apps.

#### LoRA-Tuned Dedicated Deterministic Models

LoRA fine-tuning adapts LLMs with low computational cost, enforcing stable outputs.[^26][^27]

**Implementation Steps:**

1. Fine-tune a small LLM (e.g., BERT) on prompt-output pairs for fixed mappings.

2. Encryption: Use tuned model with hashed key in prompt.

3. Decryption: Reverse prompt on same model.

**Benefits:** Reduces variance; accelerates inference by 2.3x as per studies.

**Challenges:** Initial tuning resources; use free Hugging Face tools.

**Feasibility:** Proven in papers for encryption tasks.

### Addressing Complex Key Management

#### MPC-Based Distributed Key Generation

MPC allows joint computation without revealing inputs, ideal for key creation.[^28][^29]

**Implementation Steps:**

1. Use MP-SPDZ library for parties to hold key shares.

2. Encryption: Compute prompt with share.

3. Store model locally or in encrypted cloud.

**Benefits:** Decentralizes risk; no single transmission.

**Challenges:** Overhead; limit to key exchange.

**Feasibility:** Open-source tools for enterprise encryption.

#### Steganographic Key Embedding in Chat Channels

Embed keys in LLM-generated stories for covert transfer.[^30][^31]

**Implementation Steps:**

1. Generate story embedding key fragments (e.g., acrostic).

2. Transmit as idle chat.

3. Extract and assemble.

**Benefits:** Integrates with concealment; adds pseudo-randomness.

**Challenges:** Extraction errors; add CRC checks.

**Feasibility:** Inspired by PDF guides for LLM steganography.

### Addressing Weak Security

#### FHE Hybrid Layer for Secure Mapping

Combine FHE with LLMs for computations on encrypted data.[^32]

**Implementation Steps:**

1. AES-encrypt plaintext.

2. Use FHE-LLM (Zama) to map ciphertext.

3. Decrypt with reverse FHE + AES.

**Benefits:** Mathematical security; full encryption.

**Challenges:** Speed; optimize with LoRA (6.94x speedup).

**Feasibility:** Open-source code available.

#### Differential Steganography with Multi-Layer Mapping

Generate multiple candidates, select via key for added obscurity.[^33][^34]

**Implementation Steps:**

1. LLM produces N stories, embed in one.

2. Index = hash(key) % N.

3. Extract with key.

**Benefits:** Increases attack complexity; boosts pseudo-randomness.

**Challenges:** Generation cost; parallelize small LLMs.

**Feasibility:** Supported by arXiv proofs.

### Enhancing Strengths

#### Context-Adaptive Dynamic Story Generation

Adapt outputs to conversation context for naturalness.[^35][^36]

**Implementation Steps:**

1. Include chat history in prompt.

2. Add multilingual or emoji obfuscation.

**Benefits:** Heightens concealment.

**Feasibility:** Direct LLM API use.

#### Community-Driven Model Sharing

Use Hugging Face for diverse tuned models.[^37][^38]

**Implementation Steps:**

1. Upload tuned models anonymously.

2. Use model ID as key extension.

**Benefits:** Increases randomness; community validation.

**Feasibility:** Platform-supported.

## Implementation and Feasibility Analysis

### System Architecture

The overall system comprises an LLM core, crypto modules (FHE, MPC), and interfaces for prompt engineering. Pseudocode examples:

```python
import hashlib
def generate_seed(key, event_hash):
    return hashlib.sha256(key.encode() + event_hash).hexdigest()

# Encryption
prompt = f"Encrypt '{plaintext}' with seed {seed}"
cipher = llm.generate(prompt, seed=int(seed, 16) % 2**32)
```

For FHE, use Concrete-ML:

```python
from concrete import fhe
@ fhe.compiler({"x": "encrypted"})
def fhe_map(x):
    return x + 1  # Simplified mapping
```

Feasibility is high with cloud resources; costs estimated at $0.01 per message for small models.
